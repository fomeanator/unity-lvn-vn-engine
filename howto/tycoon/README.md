# 🏪 Tycoon / менеджмент

Пошаговая экономическая игра, где каждый ход — это «месяц», а весь геймплей сводится к переменным, формулам дохода и случайным событиям.

## Что делает пример

Игрок управляет сетью кофеен. Один ход равен одному месяцу: в начале месяца считается доход, затем игрок вкладывает деньги в новую точку или в рекламу, после чего выпадает случайное событие. Цель — за `months` = 8 месяцев накопить 500💵. По истечении срока сценарий проверяет капитал и ведёт на ветку победы или проигрыша.

## Цикл хода

Главный паттерн — пошаговый цикл с меткой `:turn`, которая в начале проверяет конец игры и считает доход:

```
:turn
if month > months -> finale
// Доход месяца зависит от числа точек и репутации.
income = shops * 25 + rep * 5
cash = cash + income
```

Доход выводится по формуле `income = shops * 25 + rep * 5` — чем больше точек и репутации, тем выше прибыль. Дальше игрок выбирает инвестицию (выбор `- … -> метка`), отрабатывает случайное событие в узле `:event`, и ход замыкается:

```
:next_month
month = month + 1
-> turn
```

`month = month + 1` продвигает счётчик, а `-> turn` возвращает поток в начало цикла. Выход из цикла — единственный: когда `month > months`, прыжок уходит на `:finale`.

## Возможности движка, которые тут задействованы

- **Реактивный HUD** — строка `text` с интерполяцией `{…}` сама перерисовывается при изменении переменных:
  `text hud x=3 y=8 size=40 color=#c9f0d0 «Месяц {month}/{months}  💵{cash}  🏪{shops}  ⭐{rep}»`
- **Ходовой цикл** через метку и goto: `if month > months -> finale` на входе и `-> turn` в конце.
- **Choice-инвестиции с проверкой средств** — каждый вариант ведёт на узел, который сперва проверяет деньги: `if cash >= 120 -> do_shop`, иначе сообщение и возврат `-> turn`.
- **Случайные события** через `roll = rand(1, 6)` и каскад порогов `if roll <= 2 -> ev_good` / `if roll <= 3 -> ev_bad`.
- **Баланс через `max`** — репутация не уходит в минус: `rep = max(0, rep - 1)`.

## Разбор по шагам

**Ресурсы.** В шапке объявлены все переменные состояния:

```
cash = 100
shops = 1
rep = 0
month = 1
months = 8
```

**Панель.** Одна строка `text hud …` с интерполяцией собирает весь статус игры; править её отдельно при каждом ходе не нужно — значения подставляются автоматически.

**Цикл `:turn` — доход и выбор.** На входе проверка конца игры, затем начисление дохода и развилка инвестиций:

```
Месяц {month}. Доход составил {income}💵. Во что вложишься?
- 🏪 Новая кофейня — 120💵 -> invest_shop
- 📣 Реклама — 40💵 (+репутация) -> invest_ad
- 💰 Ничего, копить -> hold
```

**Обработчики инвестиций.** Каждый сначала проверяет средства, и только при достатке списывает деньги и меняет показатели:

```
:invest_shop
if cash >= 120 -> do_shop
Не хватает денег на новую точку.
-> turn
:do_shop
cash = cash - 120
shops = shops + 1
Открыта новая кофейня! Теперь их {shops}.
-> event
```

Реклама устроена так же (`-40💵`, `rep = rep + 2`), а вариант «копить» (`:hold`) просто идёт дальше на `:event`. Заметь: при нехватке средств поток возвращается на `-> turn` (тот же месяц), а после успешной траты — на `-> event`.

**Узел `:event` со взвешенными исходами.** Бросок кубика и каскад порогов задают вероятности: исходы `ev_good` (1–2), `ev_bad` (3) и нейтральный (4–6):

```
:event
roll = rand(1, 6)
if roll <= 2 -> ev_good
if roll <= 3 -> ev_bad
Спокойный месяц без потрясений.
-> next_month
```

`:ev_good` даёт бонус, растущий от числа точек (`bonus = 30 + shops * 10`), `:ev_bad` отнимает деньги и каплю репутации с защитой через `max`. Все три ветки сходятся на `:next_month`.

**`:next_month`** продвигает счётчик и замыкает цикл (`month = month + 1` → `-> turn`).

**Финал с проверкой.** Когда месяцы кончились, итог и развилка результата:

```
:finale
bg /content/bg/city_office.jpg
Год подошёл к концу. Капитал: {cash}💵, точек {shops}, репутация {rep}.
if cash >= 500 -> win
-> lose
```

`if cash >= 500 -> win`, иначе «провал» на `-> lose`. Обе концовки уходят на `-> __end`.

## Про баланс

Вся экономика здесь — это переменные и формулы в `.lvns`. Цены инвестиций (`120`, `40`), множители дохода (`shops * 25 + rep * 5`), размер бонуса (`30 + shops * 10`), порог победы (`500`) и вероятности событий (пороги `roll <= 2` / `roll <= 3`) — обычные числа в тексте сценария. Крутить сложность можно прямо в файле, не трогая код движка: подвинул множитель — изменился весь темп игры.

## Запуск и проверка

```sh
# собрать транскодер
cd tools/lvnconv && go build -o /tmp/lvnconv .

# скомпилировать .lvns → .lvn
/tmp/lvnconv convert -i howto/tycoon/tycoon.lvns -o /tmp/tyc.lvn

# структурная проверка: неизвестный op, висячие прыжки, дубли меток
/tmp/lvnconv validate /tmp/tyc.lvn
```

Цель — **0 warning(s)**. Самый частый ворнинг «label … reached by fall-through» лечится явным `-> метка` перед целевой меткой.

## Сделай своим

- **Новые типы вложений** — добавь вариант в развилку `:turn` и пару `invest_*`/`do_*` (например, обучение персонала, которое поднимает доход с точки).
- **Кредиты и долг** — переменная `debt` с процентом, который списывается в `:next_month`; победа только при `debt == 0`.
- **Конкурент** — переменная `rival`, которая растёт сама и режет твой `income`, пока ты не вложишься в борьбу с ней.
- **Больше событий** — расширь каскад в `:event` (увеличь диапазон `rand` и добавь ветки `if roll <= N`).
- **Несколько ресурсов/отделов** — отдельные счётчики (склад, кадры, лояльность), каждый со своей формулой и своим вкладом в общий доход.
