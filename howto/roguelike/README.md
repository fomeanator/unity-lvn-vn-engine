# 🗡 Roguelike / забег по подземелью

Процедурный спуск по подземелью: каждый забег уникален, потому что комнаты генерирует `rand`, а не сценарист.

## Что делает пример

Герой спускается из «Врат подземелья» всё глубже, комната за комнатой. Каждая комната — это случайный тип (монстр, сундук, зелье, ловушка или пусто), а между комнатами стоит риск-выбор: идти ещё глубже за добычей, выпить зелье или выйти на поверхность живым. Игрок копит глубину, золото и зелья, балансируя жадность и здоровье. Смерть (`hp <= 0`) обрывает забег без сохранения добычи — это и есть «ещё один забег».

## Сердце жанра: цикл-генератор комнат

Весь геймплей держится на одной метке-петле. `:descend` увеличивает глубину и бросает кубик типа комнаты:

```
:descend
depth = depth + 1
// Случайный тип комнаты.
room = rand(1, 6)
bg /content/bg/dungeon_room.jpg
if room <= 2 -> r_monster
if room <= 3 -> r_treasure
if room <= 4 -> r_potion
if room <= 5 -> r_trap
-> r_empty
```

Каскад `if room <= N -> ...` — это распределение вероятностей: монстр выпадает на 1–2 (чаще всего), сундук на 3, зелье на 4, ловушка на 5, а всё, что не поймал ни один `if`, проваливается в `-> r_empty`. После любой комнаты управление сходится в `:after_room`, который даёт риск-выбор «глубже / выпить зелье / выйти». А смерть стережёт на каждом уроне: `if hp <= 0 -> dead` мгновенно обрывает петлю.

## Возможности движка, которые тут задействованы

- **Реактивный HUD** — одна строка `text` с интерполяцией обновляется сама при изменении переменных:
  ```
  text hud x=3 y=8 size=42 color=#d8c9f0 «Глубина {depth}  ♥{hp}/{maxhp}  💰{gold}  🧪{potions}»
  ```
- **Цикл-генератор** — метка `:descend` с переходом `-> descend` из `:after_room` крутит бесконечную петлю комнат.
- **`rand` для всего процедурного** — тип комнаты `room = rand(1, 6)`, здоровье твари `mhp = depth * 3 + rand(0, 4)`, урон `hit = rand(2, 6)`, добыча `loot = rand(8, 20)`.
- **`chance` для развилок удачи** — `if chance(0.5) -> sneak_ok` решает, удалось ли прошмыгнуть мимо.
- **Бой-петля** — метка `:fight` бьёт себя по `-> fight`, пока кто-то не упадёт.
- **Риск-выбор** — три варианта в `:after_room` с разными последствиями.
- **`expr=` на варианте** — зелье недоступно, когда его нет: `- 🧪 Выпить зелье -> drink expr="potions >= 1"`.
- **`min` для лечения с потолком** — `hp = min(maxhp, hp + 6)` не даёт перелить выше максимума.

## Разбор по шагам

**Статы забега.** В шапке объявлены все переменные состояния — это весь «персонаж»:

```
hp = 12
maxhp = 12
gold = 0
depth = 0
potions = 0
```

**`:descend` — генерация комнаты.** Глубина растёт на 1, бросается `room = rand(1, 6)`, и каскад `if` уводит на одну из пяти меток комнат (см. выше). Это единственная точка ветвления — всё остальное стекается обратно сюда.

**Комната-монстр (бой с ответкой).** Сила твари растёт от глубины: `mhp = depth * 3 + rand(0, 4)`. Игроку дают выбор драться или прошмыгнуть:

```
:fight
hit = rand(2, 6)
mhp = mhp - hit
Ты бьёшь на {hit}.
if mhp <= 0 -> mon_dead
bite = rand(1, 4)
hp = hp - bite
Тварь огрызается на {bite}. (♥{hp})
if hp <= 0 -> dead
-> fight
```

Петля `-> fight` чередует удар игрока и ответку твари, пока `mhp <= 0` (победа → `:mon_dead`, выдаёт `drop = rand(3, 9)` золота) или `hp <= 0` (смерть → `:dead`). Альтернатива — `:sneak`: `if chance(0.5) -> sneak_ok`, иначе тварь кусает на отходе.

**Прочие комнаты.** Они короткие и сразу уходят в `:after_room`: `:r_treasure` добавляет `loot = rand(8, 20)` золота, `:r_potion` делает `potions = potions + 1`, `:r_trap` бьёт на `dmg = rand(2, 5)` (с проверкой `if hp <= 0 -> dead`), `:r_empty` — просто текст без последствий.

**`:after_room` — риск-выбор.** Сердце напряжения:

```
:after_room
- ⬇ Спуститься глубже (риск) -> descend
- 🧪 Выпить зелье -> drink expr="potions >= 1"
- 🪜 Выйти с добычей -> escape
```

«Глубже» возвращает в генератор (риск растёт), «зелье» лечит и снова показывает меню (`:drink` → `-> after_room`), «выйти» завершает забег с добычей.

**Концовки.** `:escape` — выживание с трофеями (`Глубина {depth}, золота {gold}`), `:dead` — гибель с `fade to="black"`. Обе ведут в `-> __end`.

## Почему это «ещё один забег»

Процедурность здесь не нарисована вручную, а вычисляется: `rand` решает тип комнаты, силу врага, размер добычи и урон. Реиграбельность рождается из того, что каждый проход петли `:descend` генерит новую комнату на лету — фиксированного сценария нет, есть правила и кубики. Два забега подряд никогда не совпадут, а исход определяется связкой удачи и решений игрока «жадничать или выходить».

## Сделай своим

- **Новые типы комнат.** Расширь бросок до `rand(1, 8)` и добавь ветки `if room <= N -> ...` — кузница, торговец, развилка-лестница, святилище-баф.
- **Предметы и реликвии через инвентарь.** Заведи `inv = []` и копи находки через `inv = push(inv, "амулет")`, а эффекты проверяй через `has(inv, "амулет")`.
- **Рост сложности от глубины.** Зацепка уже есть — `mhp = depth * 3 + rand(0, 4)`. Можно так же масштабировать урон ловушек и добычу.
- **Мини-босс каждые N этажей.** В `:descend` добавь `if depth >= 5 -> r_boss` (или проверку кратности) перед обычным броском комнаты.
- **Мета-прогресс между забегами.** Через `save`/`load` и переменные переноси, например, открытые реликвии или валюту прокачки из забега в забег.
