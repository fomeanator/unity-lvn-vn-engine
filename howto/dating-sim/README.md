# 💕 Dating sim / симулятор отношений

Игра про сближение с персонажами: копишь «симпатию» правильными репликами, а финал зависит от того, к кому ты стал ближе.

## Что делает пример

Действие на университетском кампусе: у игрока есть `days = 4` дня, чтобы сблизиться с одной из двух героинь — Лией или Ким. Каждый день игрок выбирает, к кому пойти, а реплики на свидании поднимают (или роняют) метр симпатии к конкретному персонажу. Дневной цикл крутится через метку `:day_start`, пока не выйдет лимит дней. В финале открываются «роуты»-концовки: позвать на крышу можно только того, к кому симпатия дошла до порога, иначе — концовка «в одиночку».

## Три кита жанра

1. **Метры симпатии на персонажа — обычные переменные.** Никакой магии: `lia` и `kim` — это просто счётчики, которые ты сам инициализируешь и меняешь:
   ```
   lia = 0
   kim = 0
   ```
   Реплики на свиданиях двигают их: `lia = lia + 2`, `kim = kim - 1`.

2. **Дневной цикл через `day`/`days` и проверку на конец.** Счётчик дня и лимит — переменные, цикл — метка, в которую мы прыгаем в конце каждого дня. Первой же командой в цикле проверяем выход:
   ```
   :day_start
   if day > days -> finale
   ```

3. **Роуты, открывающиеся по порогу, — через `expr=` на варианте выбора.** Вариант скрыт, пока выражение ложно, поэтому концовка героини доступна только при достаточной симпатии:
   ```
   - Позвать Лию -> route_lia expr="lia >= 4"
   - Позвать Ким -> route_kim expr="kim >= 4"
   - Уйти одному -> route_alone
   ```

## Возможности движка, которые тут задействованы

- **`actor_map` для двух персонажей** — связывает отображаемое имя с id каста, чтобы реплики `Лия [smile]:` управляли нужной сущностью:
  ```
  actor_map Лия=lia
  actor_map Ким=kim
  ```
- **Эмоции каста в репликах** — эмоция переключается прямо в строке реплики через `[...]`: `Лия [smile]:`, `Ким [sad]:`, `Ким [smile]:`.
- **Реактивный HUD с двумя метрами** — строка `text` с интерполяцией `{...}` сама пересчитывается, когда меняются переменные:
  ```
  text hud x=4 y=8 size=40 color=#ffd9e6 «День {day}/{days}   ❤Лия {lia}   ❤Ким {kim}»
  ```
- **Дневной цикл** — `goto` обратно в `:day_start` после каждого свидания (см. ниже).
- **`choice expr=` для гейта роутов** — фильтр-выражение прячет недоступные концовки.

## Разбор по шагам

1. **Инициализация метров и счётчика дней.** Объявляем симпатии и календарь до начала истории:
   ```
   lia = 0
   kim = 0
   day = 1
   days = 4
   ```

2. **Реактивная панель.** Один раз ставим `text hud …` с интерполяцией — дальше он сам отражает текущие `day`, `lia`, `kim`. Стартовый фон и интро ведут в цикл: `-> day_start`.

3. **Цикл `:day_start` с проверкой конца.** Сразу проверяем лимит, иначе показываем выбор «к кому пойти»:
   ```
   :day_start
   if day > days -> finale
   bg /content/bg/campus.jpg
   День {day}. С кем проведёшь время?
   - ☕ Лия в кофейне -> see_lia
   - 📚 Ким в библиотеке -> see_kim
   ```

4. **Свидания `see_lia` / `see_kim` и реплики ±симпатии.** На свидании показываем актёра, даём пару реплик-выборов и двигаем метр. «Правильный» выбор даёт больше, нейтральный — меньше, неудачный может и отнять:
   ```
   :lia_music
   lia = lia + 2
   Лия [smile]: Ты первый, кто спросил! Дай покажу плейлист…
   -> end_day
   ```
   ```
   :kim_safe
   kim = kim - 1
   Ким [sad]: Я вообще-то занимаюсь. Потом, ладно?
   -> end_day
   ```

5. **Переход дня.** Все ветки свиданий сходятся в `:end_day`, который увеличивает счётчик и возвращает в начало цикла:
   ```
   :end_day
   day = day + 1
   -> day_start
   ```

6. **Финал с гейтами роутов и fallback «в одиночку».** Когда `day > days`, попадаем в `:finale`. Варианты-роуты героинь скрыты, пока симпатия ниже порога, а вариант «уйти одному» доступен всегда — это страховочный путь, если игрок не дотянул ни до кого:
   ```
   :finale
   bg /content/bg/sunset_roof.jpg
   Семестр на исходе. Кого ты позовёшь на крышу под закат?
   - Позвать Лию -> route_lia expr="lia >= 4"
   - Позвать Ким -> route_kim expr="kim >= 4"
   - Уйти одному -> route_alone
   ```
   Каждый роут — короткая концовка, заканчивающаяся `-> __end`.

## Запуск и проверка

```sh
# собрать транскодер
cd tools/lvnconv && go build -o /tmp/lvnconv .

# скомпилировать .lvns → .lvn
/tmp/lvnconv convert -i howto/dating-sim/dating-sim.lvns -o /tmp/ds.lvn

# структурная проверка (неизвестный op, висячие прыжки, дубли меток)
/tmp/lvnconv validate /tmp/ds.lvn
```

Цель — **0 warning(s)**. Самый частый ворнинг «label … reached by fall-through» лечится явным `-> метка` или `-> __end` перед меткой-целью.

## Сделай своим

- **Третий персонаж.** Добавь `actor_map`, метр (`mei = 0`), ветку свидания и пункт выбора в `:day_start` — каркас цикла менять не придётся.
- **События-флаги.** Запоминай прошлый выбор переменной-флагом и открывай особую реплику через `expr=` (вспомни прошлое свидание только тем, кто там был).
- **Подарки/предметы.** Заведи инвентарь (`inv = []`, `inv = push(inv, "цветы")`) и давай бонус к симпатии за вручённый предмет.
- **Ревность.** Сделай, чтобы рост одной симпатии ронял другую: рядом с `lia = lia + 2` поставь `kim = kim - 1`.
- **Больше дней и мест.** Увеличь `days`, добавь новые фоны и точки свиданий — цикл `:day_start` отмасштабируется сам.
