# START HERE — движок LVN для ИИ-агента

Ты — ИИ-агент, которого попросили сделать игру на движке **LVN**. Прочитай этот
файл целиком: после него ты сможешь собрать игру любого поддерживаемого жанра
**не заглядывая в исходный код движка**. Всё, что нужно, есть в этой папке
`howto/` и в `docs/`.

---

## 1. Ментальная модель: четыре слоя

```
   ТЫ ПИШЕШЬ            ТРАНСКОДЕР           КОНТЕЙНЕР            РАНТАЙМ
  ┌──────────┐  lvnconv ┌──────────┐  грузит ┌──────────┐ играет ┌──────────┐
  │  .lvns   │ ───────► │  .lvn    │ ──────► │ рантайм  │ ─────► │  экран   │
  │ (исходник│ convert  │ (JSON,   │         │ (Unity-  │        │          │
  │  игры)   │          │  команды)│         │  пакет)  │        │          │
  └──────────┘          └──────────┘         └──────────┘        └──────────┘
        ▲                                          ▲
        │ человекочитаемый,                        │ КАСТ и АССЕТЫ берутся
        │ его и пишут                              │ из manifest.json, НЕ из .lvns
```

- **`.lvns`** — человекочитаемый исходник игры. **Это то, что ты пишешь.**
- **`.lvn`** — машинный JSON (плоский список команд). Артефакт компиляции,
  руками не правят.
- **Рантайм** — Unity-пакет `com.lvn.engine`, исполняет `.lvn`.
- **`manifest.json`** — отдельно держит **каст (персонажей), ассеты и оглавление
  глав**. ⚠ В `.lvns` персонажа определить НЕЛЬЗЯ — ты только ссылаешься на него
  по id; само определение каста живёт в манифесте (см. `CAPABILITIES.md` §7).

Игра = **данные**. Вся логика, ветвление, статы, бои, экономика описываются
командами в `.lvns`; движок их проигрывает. Это «ffmpeg для нарративных игр».

## 2. Рабочий цикл (всегда такой)

```sh
# собрать транскодер (один раз за сессию)
cd tools/lvnconv && go build -o /tmp/lvnconv .

# цикл правок:
/tmp/lvnconv convert -i path/game.lvns -o /tmp/game.lvn   # .lvns → .lvn
/tmp/lvnconv validate /tmp/game.lvn                        # структурная проверка
/tmp/lvnconv probe /tmp/game.lvn                           # краткая сводка
```

**Правило:** после каждой правки гоняй `convert` + `validate` и добивайся
`OK ... 0 warning(s)`. Валидатор ловит висячие прыжки, неизвестные команды,
дубли меток и «глава внезапно кончилась». Что именно он проверяет — `CAPABILITIES.md`
§9. Этого достаточно, чтобы проверить корректность игры **без запуска движка**;
арт можно подставлять позже.

## 3. Что можно построить (и чего нельзя)

**Можно:** визуальные/кинетические новеллы, геймбуки/CYOA, point-and-click и
квесты, RPG (статы/бой/инвентарь/уровни), dating-sim, викторины, детективы,
tycoon/менеджмент, roguelike, головоломки — всё, что управляется **кнопками +
состоянием**. Готовые разобранные примеры — папки жанров (см. ниже).

**Нельзя** (главное; полный список — `CAPABILITIES.md` §8): нет реалтайм-таймера,
нет ввода текста игроком, нет аркадной механики на таймингах. Время отмеряется
**ходами/действиями**, любой «ввод» — это `choice` или клик по `obj on_click`.

## 5. Минимальная игра целиком

```
scene hello

bg /content/bg/room.jpg
gold = 0
Незнакомец протягивает тебе монету.
- Взять -> take
- Отказаться -> refuse

:take
gold = gold + 1
Теперь у тебя {gold} золота.
-> __end

:refuse
Ты покачал головой и ушёл.
-> __end
```

Скомпилируй (`convert`) и проверь (`validate`) — получишь `OK`. Дальше наращивай:
переменные → ветвление `if` → подпрограммы `call`/`return` → реактивный HUD
`text` → постановка (`actor`/`anim`/`fade`). Каждый шаг есть в примерах жанров.

## 6. Частые ошибки агента (избегай сразу)

1. **Пытаться определить персонажа в `.lvns`.** Нельзя. Каст — в `manifest.json`
   (`sprites`) или в `cast`-блоке `.lvn`. В скрипте — только `actor <id> …`.
2. **`anim`/`move` с пробелами в кавычках в терс-форме.** `keys="…"`/`path="…"`
   (значения со пробелами) требуют legacy-форму `id=`/`prop=`. Иначе — ошибка
   компиляции. Bracket `[…]` и `to=` можно терс-формой. (`CAPABILITIES.md` §6.)
3. **Рассчитывать на `hint`.** Он no-op, баннер не рисуется. Подсказку — через
   `say`/`text`.
4. **Ждать, что `cost`/`requires_stat` сами спишут ресурс.** Нет: `cost` — просто
   подпись, гейты только показывают/скрывают вариант. Списывай явно `set`/`inc`.
5. **Надеяться на таймер реального времени.** Его нет — отмеряй ходами.
6. **Звать `ceil`.** Нет такой функции — `floor`/`round`.
7. **Проваливание в метку-цель.** Если в метку, на которую есть прыжок, ещё и
   «проваливаются» сверху — поставь `-> метка`/`-> __end` перед ней (иначе
   warning и баг «игра кончилась раньше»).
8. **Переменные с префиксом `__`.** Он служебный — не используй для своих.
9. **Считать, что отсутствующий арт — ошибка.** Нет: слой просто пропускается,
   логику видно и без графики.

## 7. Чеклист готовности игры

- [ ] `.lvns` компилируется: `convert` без ошибок.
- [ ] `validate` даёт `0 warning(s)` (или осознанно объяснённые).
- [ ] Все ветки выбора и `if` ведут на существующие метки или `__end`.
- [ ] Реактивный HUD (если есть) показывает актуальные переменные.
- [ ] Каст, который ты используешь (`actor <id>`), определён в манифесте — либо
      это осознанный греибокс (арт позже).
- [ ] Стабильные id меток/концовок не меняются (важно для save/load).
