# Язык `.lvns` — полный справочник

`.lvns` — это человекочитаемый исходник, который транскодер **lvnconv** компилирует
в контейнер `.lvn` (плоский JSON-массив команд), а его уже исполняет рантайм
(Unity-пакет `com.lvn.engine`). Писать игру нужно в `.lvns`; `.lvn` —
машинный артефакт, его не редактируют руками.

> Это **источник истины по синтаксису**. Парсер живёт в
> `tools/lvnconv/internal/lvns/convert.go`, вычислитель выражений —
> `unity/Packages/com.lvn.engine/Runtime/LvnExpression.cs`. Контейнер и команды
> описаны в `docs/lvn-format.md`; постановка — `docs/placement.md`; персонажи —
> `docs/cast.md`; анимация — `docs/animation-system.md`.

Все 12 примеров в этой папке скомпилированы и проверены этим транскодером.

## 1. Скелет файла

```
scene my-scene          // (необязательно) id сцены — служебная метка

// комментарии — двойной слэш; пустые строки игнорируются
bg /content/bg/room.jpg  // фон
Это строка повествования. // нарратив (без говорящего)
Мара: Привет.            // реплика: «Имя: текст»

- Поздороваться -> hello // выбор: каждая строка «- текст -> метка»
- Уйти -> bye

:hello                   // метка-цель прыжка
Ты помахал в ответ.
-> __end                 // переход; __end — встроенный конец сценария

:bye
-> __end
```

Главные принципы:

- **Поток сверху вниз.** Команды выполняются по порядку, кроме тех, что двигают
  курсор: `-> метка` (goto), `if`, `choice`, `call`/`return`.
- **Метки `:имя`** — единственные точки, куда можно прыгнуть. `__end` —
  встроенная метка «конец».
- **Любой неизвестный `op` — ошибка сборки**, не молчаливый пропуск. То же для
  прыжка на несуществующую метку. Гоняй `lvnconv validate` (см. ниже).

## 2. Реплики, нарратив, эмоции

| Пишешь | Что получается |
|---|---|
| `Просто текст без двоеточия.` | Нарратив (`say` без говорящего). |
| `Мара: Текст реплики.` | Реплика: `Мара` идёт на плашку имени. |
| `Мара [smile]: Текст.` | Реплика + переключение эмоции каста (ось `emotion`). |
| `«многострочный … текст»` | Французские кавычки держат текст на нескольких строках как одну логическую. |

**`actor_map Имя=id`** связывает отображаемое Имя с id персонажа в каталоге каста,
чтобы `Мара [smile]:` управляла кастом `mara`. Без маппинга id берётся как имя в
нижнем регистре с заменой пробелов на `_`.

Интерполяция: `{выражение}` в любом тексте подставляет значение —
`Золота: {gold}, атака {atk + watk}.` Литеральные фигурные скобки экранируй
удвоением: `{{` и `}}`. Несбалансированные `{`/`}` — предупреждение валидатора.

## 3. Постановка: фон, актёры, объекты

### Фон
```
bg /content/bg/room.jpg              // терс: id выводится из имени файла
bg id=room sprite_url="/path.jpg"    // legacy key=value
```

### Актёр (персонаж — гаснет, когда не говорит)
```
actor mara left smile                // терс: id, позиция, эмоция/поза
actor hero center w=.5 h=.6 x=.5 y=.5 armor={arm}
actor mara hide                      // спрятать (арт сохраняется до show)
```
Терс-форма: первый токен — id; голые слова распознаются как `hide`/`show`,
именованная позиция (`left`/`center`/`right`/`far_left`/`far_right`/
`offscreen_left`/`offscreen_right`) или иначе как значение оси `emotion`
(поза/эмоция). Пары `k=v`: `w`(→width) `h`(→height) `x` `y` `scale` `anchor` `z`
`flip` `rotation` `opacity` `on_click`, а также любые оси каста (`armor=`,
`weapon=`, …).

### Объект (всё прочее — НЕ гаснет; используется через key=value)
```
obj id=letter sprite_url="/art/letter.png" x=0.7 y=0.45 width=0.18 anchor="0.5,0.5" z=5
obj id=play   sprite_url="/ui/play.png"    x=0.5 y=0.4  anchor="0.5,0.5" on_click="start"
```

### Поля размещения (доли экрана, 0..1)
| Поле | Смысл |
|---|---|
| `position` | Именованный слот по горизонтали (заменяет `x`). |
| `x`,`y` | Где стоит точка привязки объекта (0,0 — левый верх; 1,1 — правый низ). По умолчанию `y`=1 (низ экрана). |
| `width`,`height` | Размер в долях экрана. По умолчанию 0.46 × 0.62. |
| `anchor` | Своя точка привязки `"ax,ay"`. По умолчанию `"0.5,1"` (низ-центр, «на полу»). `"0.5,0.5"` — центр. |
| `z` | Порядок отрисовки; больше — ближе. |
| `flip` | `true` — отзеркалить. |
| `rotation` | Градусы. `opacity` — 0..1. `show` — `false` прячет. |

### `on_click` → кликабельные игры
Дай объекту `on_click="метка"` — он станет хотспотом: клик прыгает на метку и
**гасит тап** (диалог не проматывается). «Экран» — это пауза (нарратив-строка),
вокруг которой расставлены хотспоты, а метка возвращает обратно. Этого +
`goto`/`if`/переменных хватает на point-and-click, меню, гeймбук, hidden-object.

В контейнере `.lvn` у `on_click` есть и объектная форма с попутной установкой
переменных: `"on_click": { "goto": "метка", "set": { "flag": 1 } }` (валидатор
проверяет `goto` внутри неё как обычную цель прыжка).

## 4. Эффекты, звук, тайминг

| Команда | Пример | Смысл |
|---|---|---|
| `fade` | `fade to="black" duration=0.8` | Полноэкранное затемнение (`black`/`white`/`clear`). |
| `dim` | `dim alpha=0.6 duration=0.5` | Притушить сцену (фокус). `alpha=0` — вернуть. |
| `flash` | `flash to="white" duration=0.3` | Вспышка. |
| `tint` / `blur` | `tint …` `blur …` | Цветовой фильтр / размытие. |
| `camera` | `camera action=shake amplitude=0.02 duration=0.4` | `shake`/`zoom`/`pan`. |
| `particles` | `particles type=rain on=true` | Слой частиц (`rain`/`snow`/…), `on=false` выключить. |
| `audio` | `audio channel=music action=play url="/a.ogg"` | Каналы `music`/`sfx`/`ambient`; `play`/`stop`/… |
| `wait` | `wait ms=500` | Пауза перед следующей командой. |
| `hint` | `hint text="…" show=true` | ⚠ **No-op в эталонном рантайме** — баннер не рисуется. Команда валидна, но не полагайся на неё; подсказку давай через `say`/`text`. |
| `preload` | `preload assets=…` | Подсказка загрузчику прогреть ассеты. |
| `text_pace` | `text_pace cps=40` | Скорость печати (символов/сек; `0` — мгновенно). |

## 5. Поток управления

### Переходы и развилки
```
-> метка                 // безусловный прыжок (goto)
if cond -> метка         // если истина — прыжок, иначе падаем дальше
```

### Блочные `if / else`
```
if gold >= 10 {
  gold = gold - 10
  Куплено.
} else {
  Не хватает.
}
```
Однострочно тоже можно: `if c { … } else { … }`. Скобка `{` должна
завершать открывающую строку (или используй однострочную форму, её транскодер
сам разложит).

### Выбор
```
- Текст варианта -> метка
- С параметрами -> метка cost="3 хода" expr="gold >= 5"
```
Параметры варианта: `cost=` (только подпись «цены» — **сама ничего не списывает**,
ресурсы убавляй явно на метке-обработчике), `requires_stat`/`min` (порог —
вариант скрыт, если переменная < min), `expr=` (булев фильтр — **вариант скрыт,
когда выражение ложно**). В контейнере `.lvn` у варианта вместо `goto` может быть
`body` — встроенный список команд, исполняемый при выборе (только `set`/`inc`/
постановочные команды и `goto`; **без** `if`/`choice`/`call` внутри — сложную
логику выноси на отдельную метку).

### Циклы
```
for it in inv {          // обход коллекции
  Предмет: {it}.
}

while xp >= need {        // пока условие истинно
  xp = xp - need
  level = level + 1
}
```

### Подпрограммы
```
call fight               // прыжок с запоминанием точки возврата
return                   // вернуться после соответствующего call
```

### Функции (сахар над call/return)
```
func show_hero() {
  actor hero left armor={arm} weapon={wpn}
}
func add(a, b) {
  return a + b
}

show_hero()              // вызов
sum = add(2, 3)          // вызов с возвратом значения
```
Аргументы привязываются к параметрам **позиционно**. Возврат значения — через
`return <выражение>`.

### Сохранение/загрузка
```
save                     // снимок состояния
load                     // восстановить снимок
```

## 6. Переменные и состояние

```
gold = 12                // присваивание (и объявление, и мутация)
gold = gold - 6
name = "Мара"
inv = []                 // пустой список
flags = {}               // пустая карта (через builder, см. ниже)
```

- Любая **необъявленная переменная читается как `0`/`""`/`false`** — но явная
  инициализация делает сценарий понятнее.
- **Служебный префикс `__`** зарезервирован за авто-переменными транскодера/
  рантайма (`__ret`, `__seen_*`, `__i1` …). Не называй так свои переменные.
- Точечные имена допустимы: `ns.flag` — для неймспейсинга глобалок.
- **Типы значений:** число, строка, bool, `null`, список `[]`, карта `{}`.
- В контейнере `.lvn` `if` принимает не только строковый `expr`, но и
  **структурный** `cond` — `{ "key": "courage", "op": "gte", "value": 2 }`
  (`eq/ne/lt/lte/gt/gte`); `set` — `value` (литерал) или `expr` (выражение,
  главнее); `inc` — `by` (по умолчанию `+1`). В `.lvns` ты обычно пишешь короткую
  форму (`gold = gold + 1`, `if gold >= 2 -> …`), а транскодер выдаёт нужный вид.

## 7. Выражения

Операторы: `+ - * /`, сравнения `== != > >= < <=`, логика `&& || !`, скобки.
Строковые литералы в кавычках. Внутри `«…»`-текста экранируй кавычки как `\"`
(например `expr="has(inv, \"ключ\")"`).

### Встроенные функции (вычислитель — `LvnExpression.cs`)

**Числа и случайность**
| Функция | Результат |
|---|---|
| `rand()` | дробное 0..1 |
| `rand(n)` | целое 0..n включительно |
| `rand(a, b)` | целое a..b включительно |
| `chance(p)` | `true` с вероятностью p (по умолчанию 0.5) |
| `min(a,b)` `max(a,b)` | минимум / максимум |
| `abs(x)` `floor(x)` `round(x)` | модуль / вниз / округление |

**Чтение коллекций**
| Функция | Результат |
|---|---|
| `len(x)` | длина списка/карты/строки |
| `has(coll, x)` | есть ли элемент / ключ / подстрока |
| `get(coll, key[, default])` | безопасное чтение |
| `indexof(arr, x)` | индекс или `-1` |
| `count(arr, x)` | сколько раз встречается |
| `sum(arr)` | сумма чисел |
| `first(arr)` `last(arr)` | первый / последний |
| `keys(obj)` `vals(obj)` | ключи / значения карты |

**Сборка коллекций** (возвращают НОВОЕ значение — присваивай обратно: `inv = push(inv, x)`)
| Функция | Результат |
|---|---|
| `list(a, b, …)` | новый список |
| `push(arr, x)` | список + элемент в конец |
| `pop(arr)` | список без последнего |
| `removeat(arr, i)` | список без элемента с индексом i |
| `remove(arr, x)` | список без первого равного x |
| `slice(arr, s[, e])` | срез |
| `concat(a, b, …)` | склейка списков |
| `put(map, key, val)` | карта с установленным ключом |
| `del(map, key)` | карта без ключа |

> Примечание: `ceil` нет — используй `floor(x)` или `round(x)`.

## 8. Каст — параметрические персонажи

Персонаж — это **именованная сущность** с **осями** (поза, эмоция, одежда…), а
его арт — набор **шаблонов-слоёв** с токенами `{ось}`. Чтобы показать персонажа
в любом состоянии, называешь сущность и значения осей; рантайм подставляет
токены и накладывает слои снизу вверх.

```json
"cast": {
  "mara": {
    "name": "Мара",
    "layers": [
      "/art/mara/body_{pose}.png",
      "/art/mara/face_{emotion}.png",
      "/art/mara/{prop}.png"
    ],
    "defaults": { "pose": "stand", "emotion": "neutral" }
  }
}
```

- Токен разрешился → слой рисуется; токен без значения → слой **пропускается**
  (поэтому необязательные части вроде `{prop}` появляются только по запросу).
- **K поз + M эмоций = K + M картинок, а не K × M** — экономия из шаблонов.
- В сценарии путей нет: `actor mara emotion=smile pose=arms prop=umbrella`.

⚠ **В `.lvns` каст определить НЕЛЬЗЯ** — там нет каст-директивы. Скрипт только
**ссылается** на персонажа по id (`actor mara …`). Само определение живёт в
**`manifest.json` → `sprites`** (глобально для всех глав) или в `cast`-блоке
`.lvn` (локально). Создавать/править персонажей удобно в визуальном редакторе
каста (панель IDE), который пишет в `manifest.json`. Полный пайплайн каста и
ассетов (резолвинг, добавление арта, пути, заглушки) — [`CAPABILITIES.md`](CAPABILITIES.md)
§7; модель — `docs/cast.md`.

## 9. Анимация — `anim` и `move`

Одно правило: **разные каналы идут параллельно, ключи внутри канала — по очереди.**
Два глагола, оба компилируются в одну рантайм-команду `anim`.

```
// Одной строкой (твин от текущего значения к цели):
anim mara scale to=1.1 dur=0.4 ease=outBack
move id=mara to=0.2,-0.05 dur=1 ease=inOutSine

// Терс-формы: bracket-список значений, растянутый по dur:
anim mara scale [1 1.03 1] 3s yoyo

// Ключевые кадры «время:значение» — нужна legacy-форма id=/prop=/keys=
// (терс-форма ломается на пробелах внутри кавычек):
anim id=mara prop=rotation keys="0:0 1:8 2:-8 3:0" loop=yoyo ease=inOutSine
move id=mara path="-0.18,0.04 0,-0.03 0.18,0.04" dur=2 ease=outCubic

// Остановить script-анимации сущности:
anim mara stop
```

Свойства (`prop`): `x` `y` (сдвиг в долях своего размера) · `screen_x` `screen_y`
(движение по экрану) · `scale` `scalex` `scaley` · `rotation` · `alpha` · `frame`
(смена кадра слоя по оси). Easing: `linear` `inOutSine` `outCubic` `outBack`
`inBack`. `loop`: `once`(по умолч.) / `true`(restart) / `yoyo`. Параллельность —
просто две строки `anim`. Циклящаяся анимация **никогда не блокирует** сценарий.

Правило кавычек: значения со **пробелами** (`keys="…"`, `path="…"`) требуют
legacy-форму с `id=`/`prop=`. Bracket-список `[…]` и one-liner `to=` работают и
в терс-форме. Полная спека и статусы фич — `docs/animation-system.md`.

## 10. Сборка и проверка

```sh
# собрать транскодер
cd tools/lvnconv && go build -o /tmp/lvnconv .

# скомпилировать .lvns → .lvn
/tmp/lvnconv convert -i howto/visual-novel/visual-novel.lvns -o /tmp/out.lvn

# структурная проверка (неизвестный op, висячие прыжки, дубли меток)
/tmp/lvnconv validate /tmp/out.lvn          # -strict: ворнинги = ошибки

# краткая сводка по .lvn
/tmp/lvnconv probe /tmp/out.lvn
```

Гоняй `validate` после каждой правки. Хорошая цель — **0 warning(s)**:
самый частый ворнинг «label … reached by fall-through» означает, что в метку-цель
прыжка ещё и «проваливаются» сверху — поставь явный `-> метка` или `-> __end`
перед ней (см. идиому в примерах).
